# Pathwise Buyer Demo Script (7 minutes)

## Goal
Show that Pathwise delivers **real-time, actionable support** for neurodiverse visitors — not generic itinerary text.

## Demo setup (before call)
- Open app home page
- Keep a prepared venue URL and a prepared profile
- Keep one generated plan with transport + live dashboard available

---

## 0:00–1:00 — Problem framing
"Most accessibility tools are static checklists. Pathwise gives people in-the-moment support before and during a real trip."

## 1:00–2:00 — Create a plan quickly
1. Go to onboarding
2. Highlight sensitivity settings + detail preference
3. Paste venue URL and generate plan

Call out:
- Sensory-aware sections
- Risk summary + risk breakdown
- Live updates + practical tips
- Weather/transport context

## 2:00–4:00 — Real-time support differentiator
Open generated plan and show:
- Live Trip Dashboard buttons:
  - I have arrived
  - Stop approaching
  - Near loud area / food court
  - Queue too long
  - Find quiet exit
  - 90-second reset

Narration:
"This is where Pathwise becomes operational during the trip, not just pre-trip."

## 4:00–5:30 — Safety + collaboration
Show:
- Overwhelmed plan
- Community tips moderation model
- Share/collaborate workflow (trusted supporter)
- Print/PDF social story option

## 5:30–6:30 — Operations & readiness
Show `/setup` and mention:
- Runtime diagnostics
- Graceful fallbacks when external APIs are unavailable
- Current test/build status

## 6:30–7:00 — Commercial close
"Ideal pilot: one institution, one cohort, 4 weeks. Success metrics: completion rate, reduced distress events, user confidence scores."

---

## Objection handling

### "Is this production ready?"
- Yes for pilot use with current baseline (build/lint/tests passing).
- Full commercial go-live follows the checklist in `docs/GO_LIVE_CHECKLIST.md`.

### "What’s unique vs generic AI?"
- Structured accessibility extraction
- In-the-moment dashboard prompts
- Transport + sensory + risk in one flow
- Printable social story with practical support cues

### "How do we trial this?"
- Pilot deployment + branded onboarding
- Instrumented metrics from day one
- Weekly progress report against agreed KPIs
